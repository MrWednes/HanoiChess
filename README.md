# HanoiChess
A board game based on Tower of Hanoi
一个奇怪甚至可能愚蠢的命令行棋类游戏，受到汉诺塔移动规则的启发。
整个游戏完全由我自己设计，自己实现。

### 游戏描述及规则

**设定介绍**：
1.游戏有黑白两方两个玩家进行
2.游戏由黑白棋子各5枚，和5x5的方形棋盘组成
3.棋子有大小之分，五个等级从小到大分别为1，2，3，4，5
4.白方拥有白色棋子，黑方拥有黑色棋子
5.开始阶段黑色棋子和白色棋子分别大到小叠成一摞，分别置于棋盘的左上角和右下角，如下图。
游戏目标：将己方的五枚棋子早于对方移入对方的棋子起始处，棋子移入该位置时会从棋盘上消失。

**棋子移动规则：**
1.玩家双方轮流行棋，只可移动己方棋子，每次行棋只可移动一枚棋子一步，一步为棋子所在网格交点所在方格的任意一个处自身所在交点的任意一个交点。
2.较小的棋子可以放置到较大的棋子上方，这时下方棋子不可移动，较大棋子不可放置到较小棋子上。
3.当对方较小棋子防止到我方较大棋子上时，我方可用与对方相同大小的棋子，与对方的棋子调换位置。

### 代码实现
1.本代码通过命令行的形式对棋盘进行了模拟，如下图
2.*占位符表示了棋盘的交点，棋子的颜色和大小分别由字母b, w和数字1-5表示
3.棋子的上下叠加由从左到右的顺序代替
4.棋盘上边和右边分别标由棋盘交点的坐标
5.玩家通过在命令行输入棋子起始点坐标控制棋子移动
6.某方玩家移动完所有棋子，游戏结束显示胜方玩家
7.我同时编写了机器玩家，有三种模式可供选择

### 代码设计细则
面向对象编程，共有11各类构成，其职责分别如下
HanoiChess 汉诺棋类：程序主入口，负责游戏初始化，模式选择，交替行棋。
Board 棋盘类：负责交点的存储，棋盘的展示
Cross 交点类：负责棋子的存储
CrossEnd 交点终点类：继承自Cross，负责检验进入的棋子是否合法，负责记分。
Piece 棋子类 ：大小和颜色
IPlayer 抽象类 玩家：负责移动棋子。
Player 玩家类继承自IPlayer：从命令行接收移动指令
ComputerGreedy :继承自IPlayer, 基于最大棋子优先移动的原则设计的机器走棋
Rule 规则类：负责判断移动棋子是否合法，提示不合法的移动，游戏结果
RuleForDroid: 负责判断机器的尝试移动是否合法
StepPriority : 存储棋子的移动优先级，用于机器移动棋子
